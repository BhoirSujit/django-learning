# Django Models: In-Depth Guide

## Introduction

Django Models define the structure of your database.  
They are written using Python classes and represent database tables.

Model = Blueprint of a database table

Mapping:

- Model class → Database table  
- Field → Column  
- Object → Row  

Django uses ORM (Object Relational Mapping) to convert Python code into SQL automatically.

---

## How Models Work Internally

When you define:

```python
class Student(models.Model):
    name = models.CharField(max_length=100)
```

Django:

1. Detects model changes
2. Creates a migration file
3. Converts model to SQL
4. Creates or updates the database table
5. Maps Python objects to database rows

Simplified SQL:

```sql
CREATE TABLE student (
    id INTEGER PRIMARY KEY,
    name VARCHAR(100)
);
```

---

## Basic Model Example

```python
from django.db import models

class Student(models.Model):
    name = models.CharField(max_length=100)
    age = models.IntegerField()
    email = models.EmailField(unique=True)
    created_at = models.DateTimeField(auto_now_add=True)

    def __str__(self):
        return self.name
```

Django automatically creates:

- `id` field as Primary Key
- Table name: `appname_student`

- auto_now=True → Updates every save()

## Field Options

```python
name = models.CharField(max_length=100, null=True, blank=True)
```

Common options:

- max_length → Required for CharField
- null=True → Database can store NULL
- blank=True → Form validation allows empty
- default=value → Default value
- unique=True → Unique constraint
- db_index=True → Adds database index
- choices=[...] → Restrict values

Difference:

- null → Database level
- blank → Form level

## Model Relationships

### 1. ForeignKey (One-to-Many)

```python
class Author(models.Model):
    name = models.CharField(max_length=100)

class Book(models.Model):
    title = models.CharField(max_length=200)
    author = models.ForeignKey(
        Author,
        on_delete=models.CASCADE,
        related_name="books"
    )
```

If Author is deleted → Books are deleted (CASCADE).

Reverse query:

```python
author.books.all()
```

---

### 2. ManyToManyField

```python
class Course(models.Model):
    name = models.CharField(max_length=100)

class Student(models.Model):
    name = models.CharField(max_length=100)
    courses = models.ManyToManyField(Course)
```

Django creates an intermediate table automatically.

---

### 3. OneToOneField

```python
from django.contrib.auth.models import User

class Profile(models.Model):
    user = models.OneToOneField(User, on_delete=models.CASCADE)
    bio = models.TextField()
```

Used to extend the built-in User model.

## Meta Class

```python
class Student(models.Model):
    name = models.CharField(max_length=100)

    class Meta:
        ordering = ['name']
        verbose_name = "Student"
        verbose_name_plural = "Students"
        db_table = "custom_student_table"
        indexes = [
            models.Index(fields=['name']),
        ]
```

Used for:
- Default ordering
- Custom table name
- Human-readable names
- Database indexing

## Custom Model Methods

```python
class Student(models.Model):

    def is_adult(self):

```

## Custom Model Managers

```python
class AdultManager(models.Manager):
    def get_queryset(self):
        return super().get_queryset().filter(age__gte=18)

class Student(models.Model):
    age = models.IntegerField()
    adults = AdultManager()
```

Usage:

```python
Student.adults.all()
```

## Abstract Base Models

Reusable base model.

```python
class TimeStampedModel(models.Model):
    created_at = models.DateTimeField(auto_now_add=True)
    updated_at = models.DateTimeField(auto_now=True)

    class Meta:
        abstract = True
```

No separate table is created.

---

## Proxy Models

```python
class StudentProxy(Student):
    class Meta:
        proxy = True
        ordering = ['-age']
```

No new table created. Only behavior changes.

## Migrations

After creating or modifying models:

```bash
python manage.py makemigrations
python manage.py migrate
```

- makemigrations → Creates migration file
- migrate → Applies changes to database

Migration files are stored inside:

```
app/migrations/
```

## ORM Operations

### Create

```python
Student.objects.create(name="Sujit", age=22)
```

### Retrieve

```python
Student.objects.all()
Student.objects.filter(age__gte=18)
Student.objects.get(id=1)
```

### Update

```python
student = Student.objects.get(id=1)
student.age = 23
student.save()
```

### Delete

```python
student.delete()
```

## Query Optimization

Avoid N+1 problem.

Bad:

```python
for book in Book.objects.all():
    print(book.author.name)
```

Better:

```python
Book.objects.select_related('author')
```

For ManyToMany:

```python
Book.objects.prefetch_related('categories')
```

---

## How Django Prevents SQL Injection

Django ORM automatically parameterizes queries.

```python
Student.objects.filter(name=user_input)
```

User input is safely escaped.

- Keep models clean and simple
- Always run migrations after changes
