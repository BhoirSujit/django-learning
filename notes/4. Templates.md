# Django Templates: In-Depth Guide

## Introduction

Django Templates are used to create the frontend (HTML pages) of your web application. They allow you to display dynamic data from Python in a clean and organized way.

Templates separate logic from design, making projects easier to manage.

---

## What is a Template?

A template is an HTML file that contains special Django syntax to display data.

Example:

```html
<h1>Hello {{ name }}</h1>
```

Here, `{{ name }}` is replaced by real data at runtime.

---

## Why Use Templates?

Templates help you:

* Keep Python and HTML separate
* Reuse UI components
* Make dynamic websites
* Maintain clean code

Without templates, your HTML would be mixed with Python logic.

---

## Template Folder Structure

Recommended structure:

```
project/
├── app/
│   └── views.py
└── templates/
    └── app/
        └── home.html
```

Best practice: Create app-wise folders inside templates.

---

## Configure Templates in settings.py

Open `settings.py` and check:

```python
TEMPLATES = [
    {
        'BACKEND': 'django.template.backends.django.DjangoTemplates',
        'DIRS': [BASE_DIR / 'templates'],
        'APP_DIRS': True,
    },
]
```

This tells Django where to find templates.

---

## Rendering Templates in Views

Example view:

```python
from django.shortcuts import render

def home(request):
    return render(request, 'app/home.html')
```

Using `render()` loads HTML and sends response.

---

## Passing Data to Templates (Context)

Context = data sent from view to template.

Example:

```python
def home(request):
    data = {
        'name': 'Sujit',
        'age': 22,
    }
    return render(request, 'app/home.html', data)
```

In template:

```html
<p>Name: {{ name }}</p>
<p>Age: {{ age }}</p>
```

## Django Template Language (DTL)

Django uses its own template language.

### 1. Variables

```html
{{ variable }}
```

---

### 2. If Condition

```html
{% if age >= 18 %}
  <p>Adult</p>
{% else %}
  <p>Minor</p>
{% endif %}
```

---

### 3. For Loop

```html
{% for item in items %}
  <li>{{ item }}</li>
{% endfor %}
```

---

### 4. Comments

```html
{# This is a comment #}
```

---

## Template Inheritance (Very Important)

Used to reuse layouts.

### Base Template (base.html)

```html
<!DOCTYPE html>
<html>
<head>
  <title>{% block title %}My Site{% endblock %}</title>
</head>
<body>
  {% block content %}
  {% endblock %}
</body>
</html>
```

### Child Template

```html
{% extends 'base.html' %}

{% block title %}Home{% endblock %}

{% block content %}
<h1>Welcome</h1>
{% endblock %}
```

Inheritance avoids duplicate code.

---

## Include Templates

Used to reuse small components.

```html
{% include 'navbar.html' %}
```

Example uses: headers, footers, menus.

---

## Static Files in Templates

Static files = CSS, JS, Images.

### Load static

```html
{% load static %}
```

### Use static

```html
<link rel="stylesheet" href="{% static 'css/style.css' %}">
```

---

## Filters in Templates

Filters modify data.

Example:

```html
{{ name|upper }}
{{ text|truncatewords:5 }}
```

Common filters:

* upper
* lower
* length
* date
* safe

---

## Template Tags

Tags control logic.

Examples:

* {% if %}
* {% for %}
* {% block %}
* {% include %}
* {% url %}

Example:

```html
<a href="{% url 'home' %}">Home</a>
```

---

## Using URLs in Templates

In urls.py:

```python
path('', home, name='home')
```

In template:

```html
{% url 'home' %}
```

Prevents hardcoding links.

---

## Template Security

Django automatically escapes HTML.

Example:

```html
{{ user_input }}
```

Prevents XSS attacks.

Use `|safe` only when trusted.

---

## Common Template Errors

### VariableDoesNotExist

* Wrong variable name

### Invalid Block Tag

* Syntax error
