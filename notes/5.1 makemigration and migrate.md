# Difference Between makemigrations and migrate in Django

Framework: Django (Python Web Framework)

---

## Overview

In Django, database changes happen in two steps:

1. Create migration file
2. Apply migration to database

These steps are handled by:

- `makemigrations`
- `migrate`

---

## 1️⃣ python manage.py makemigrations

### What It Does

- Detects changes in `models.py`
- Creates a migration file
- Does NOT modify the database

It prepares the changes but does not execute them.

---

### Example

If you modify your model:

```python
class Student(models.Model):
    name = models.CharField(max_length=100)
    age = models.IntegerField()
```

Run:

```bash
python manage.py makemigrations
```

Django creates a file like:

```
0002_add_age_field.py
```

Location:

```
app/migrations/
```

This file contains instructions for updating the database.

---

## 2️⃣ python manage.py migrate

### What It Does

- Reads migration files
- Executes SQL queries
- Updates the actual database schema

It applies the prepared changes to the database.

---

### Example

After running:

```bash
python manage.py migrate
```

The database table is updated with the new `age` column.

---

## Step-by-Step Flow

1. Modify model
2. Run `makemigrations` → Migration file created
3. Run `migrate` → Database updated

---

## Real-Life Analogy

| Command        | Meaning                     |
|---------------|-----------------------------|
| makemigrations | Write instructions          |
| migrate        | Execute instructions        |

---

## Important Notes

- Running only `makemigrations` → No database change
- Running `migrate` without new migrations → Nothing happens
- Both commands are required for database updates

---

## Interview Explanation

- `makemigrations` generates migration files based on model changes.
- `migrate` applies those migration files to the database schema.

---

## Summary

| Feature          | makemigrations | migrate |
|------------------|---------------|---------|
| Detect model changes | Yes | No |
| Create migration file | Yes | No |
| Modify database | No | Yes |
| Executes SQL | No | Yes |

---

makemigrations → Prepare changes  
migrate → Apply changes  

Both are required for updating the database properly.
