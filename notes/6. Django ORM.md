# Django ORM: In-Depth Guide

Framework: Django (Python Web Framework)

---

# 1️⃣ What is Django ORM?

ORM = Object Relational Mapping

Django ORM allows you to interact with the database using Python code instead of writing SQL queries.

Mapping:

- Python Class → Database Table
- Model Field → Column
- Object → Row

Example:

```python
Student.objects.all()
```

Equivalent SQL:

```sql
SELECT * FROM student;
```

No raw SQL required.

---

# 2️⃣ ORM Fundamentals

Every model automatically gets a manager:

```python
objects = models.Manager()
```

This allows database queries like:

```python
Student.objects.all()
Student.objects.filter(age=22)
Student.objects.get(id=1)
```

QuerySets are:

- Lazy (not executed until needed)
- Chainable
- Immutable

Example:

```python
qs = Student.objects.filter(age__gte=18)
```

Query not executed until evaluated.

# 3️⃣ CRUD Operations

## Create

```python
Student.objects.create(name="Sujit", age=22)
```

Or:

```python
student = Student(name="Rahul", age=20)
student.save()
```

---

## Read

```python
Student.objects.all()
Student.objects.get(id=1)
Student.objects.filter(age=22)
```

---

## Update

```python
student = Student.objects.get(id=1)
student.age = 23
student.save()
```

Direct update:

```python
Student.objects.filter(age=22).update(age=23)
```

---

## Delete

```python
student.delete()
```

Or:

```python
Student.objects.filter(age__lt=18).delete()
```

# 4️⃣ Filtering Data

Basic filtering:

```python
Student.objects.filter(age=22)
```

Field lookups:

- exact
- contains
- icontains
- gt (greater than)
- gte (greater than equal)
- lt (less than)
- lte (less than equal)
- in
- range

Examples:

```python
Student.objects.filter(age__gte=18)
Student.objects.filter(name__icontains="su")
Student.objects.filter(age__range=(18, 25))
Student.objects.filter(id__in=[1,2,3])
```

Exclude:

```python
Student.objects.exclude(age=22)
```

---

# 5️⃣ Logical Operators (AND, OR, NOT)

Use Q objects:

```python
from django.db.models import Q
```

## AND (default)

```python
Student.objects.filter(age=22, name="Sujit")
```

## OR

```python
Student.objects.filter(
    Q(age=22) | Q(name="Rahul")
)
```

## NOT

```python
Student.objects.filter(~Q(age=22))
```

Complex example:

```python
Student.objects.filter(
    Q(age__gte=18) & (Q(name__icontains="s") | Q(email__icontains="gmail"))
)
```

# 6️⃣ Selective Columns

Retrieve specific fields only.

## values()

Returns dictionary:

```python
Student.objects.values("name", "age")
```

Output:

```python
{"name": "Sujit", "age": 22}
```

## values_list()

Returns tuple:

```python
Student.objects.values_list("name", "age")
```

Single column flat:

```python
Student.objects.values_list("name", flat=True)
```

## only()

Loads only specific fields:

```python
Student.objects.only("name")
```

## defer()

Excludes specific fields:

```python
Student.objects.defer("email")
```

# 7️⃣ Aggregation Functions

Used to perform calculations.

Import:

```python
from django.db.models import Count, Sum, Avg, Max, Min
```

## Count

```python
Student.objects.aggregate(Count("id"))
```

## Sum

```python
Student.objects.aggregate(Sum("age"))
```

## Average

```python
Student.objects.aggregate(Avg("age"))
```

## Max / Min

```python
Student.objects.aggregate(Max("age"))
Student.objects.aggregate(Min("age"))
```

---

## Group By (annotate)

```python
Student.objects.values("age").annotate(total=Count("id"))
```

Equivalent SQL:

```sql
SELECT age, COUNT(id) FROM student GROUP BY age;
```

Descending:

```python
Student.objects.order_by("-age")
```

Multiple fields:

```python
Student.objects.order_by("age", "-name")
```

# 9️⃣ Bulk Create

Insert multiple records efficiently.

```python
students = [
    Student(name="A", age=20),
    Student(name="B", age=21),
    Student(name="C", age=22),
]

Student.objects.bulk_create(students)
```

Advantages:

- Single query
- Faster performance
- Useful for large datasets

---

# 🔟 Bulk Update

```python
students = Student.objects.filter(age__lt=18)

for s in students:
    s.age = 18

Student.objects.bulk_update(students, ["age"])
```

# 1️⃣1️⃣ Query Optimization

Avoid N+1 problem.

Bad:

```python
for book in Book.objects.all():
    print(book.author.name)
```

Good:

```python
Book.objects.select_related("author")
```

For ManyToMany:

```python
Book.objects.prefetch_related("categories")
```

---

# 1️⃣2️⃣ Lazy Evaluation

QuerySets are lazy.

Query runs only when:

- Iterated
- Printed
- Converted to list
- Sliced
- Counted

Example:

```python
qs = Student.objects.filter(age=22)
```

No query yet.
