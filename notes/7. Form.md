# Django Forms: In-Depth Guide

Framework: Django (Python Web Framework)

---

# 1️⃣ What Are Django Forms?

Django Forms are used to:

- Handle user input
- Validate data
- Clean data
- Render HTML form fields
- Protect against security issues

They solve 3 main problems:

1. Validation
2. Security
3. Data cleaning

---

# 2️⃣ Types of Forms in Django

There are two main types:

1. forms.Form
2. forms.ModelForm

---

# 3️⃣ Basic Form (forms.Form)

Example:

```python
from django import forms

class StudentForm(forms.Form):
    name = forms.CharField(max_length=100)
    age = forms.IntegerField()
    email = forms.EmailField()
```

This form:

- Generates HTML input fields
- Validates input automatically
- Provides error handling

---

# 4️⃣ Using Form in View (Full Flow)

```python
from django.shortcuts import render
from .forms import StudentForm

def student_form_view(request):
    if request.method == "POST":
        form = StudentForm(request.POST)

        if form.is_valid():
            name = form.cleaned_data["name"]
            age = form.cleaned_data["age"]
            email = form.cleaned_data["email"]

            # Process data (save to DB, send email, etc.)
    else:
        form = StudentForm()

    return render(request, "form.html", {"form": form})
```

# 5️⃣ Form Validation (Deep Understanding)

Django automatically validates:

- Required fields
- Data types
- Max length
- Email format

Validation process:

1. Field-level validation
2. clean_<fieldname>() method
3. clean() method (form-wide validation)

---

## Field-Level Validation

```python
class StudentForm(forms.Form):
    age = forms.IntegerField()

    def clean_age(self):
        age = self.cleaned_data["age"]
        if age < 18:
            raise forms.ValidationError("Age must be 18 or above")
        return age
```

---

## Form-Wide Validation

```python
class StudentForm(forms.Form):
    password = forms.CharField(widget=forms.PasswordInput)
    confirm_password = forms.CharField(widget=forms.PasswordInput)

    def clean(self):
        cleaned_data = super().clean()
        password = cleaned_data.get("password")
        confirm = cleaned_data.get("confirm_password")

        if password != confirm:
            raise forms.ValidationError("Passwords do not match")
```

# 6️⃣ cleaned_data

After validation:

```python
form.cleaned_data
```

Returns dictionary of validated data.

Example:

```python
{
    "name": "Sujit",
    "age": 22
}
```

Only available after `form.is_valid()`.

# 7️⃣ ModelForm (Very Important)

ModelForm connects form directly with database model.

Example model:

```python
class Student(models.Model):
    name = models.CharField(max_length=100)
    age = models.IntegerField()
    email = models.EmailField()
```

ModelForm:

```python
from django.forms import ModelForm

class StudentForm(ModelForm):
    class Meta:
        model = Student
        fields = ["name", "age", "email"]
```

Django automatically:

- Creates fields
- Adds validation
- Connects to database

---

# 8️⃣ Saving ModelForm

```python
form = StudentForm(request.POST)

if form.is_valid():
    form.save()
```

---

## Save Without Commit

```python
student = form.save(commit=False)
student.age = 25
student.save()
```

Used when modifying data before saving.

# 1️⃣7️⃣ Forms vs ModelForms

| Feature | Form | ModelForm |
|----------|------|-----------|
| Linked to DB | ❌ | ✅ |
| Auto-save | ❌ | ✅ |
| Manual fields | ✅ | Optional |
| Use case | Custom logic | Database models |

# 1️⃣8️⃣ Best Practices

- Always use `is_valid()` before accessing data
- Use ModelForm for database models
- Add custom validation when needed
- Keep form logic clean
- Use widgets for better UI
- Always include CSRF token
